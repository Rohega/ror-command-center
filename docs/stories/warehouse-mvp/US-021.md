# US-021: Lista de picking

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-020

---

## Objetivo

Generar una lista de trabajo ordenada por ubicación para que el picker recorra el almacén de forma eficiente al preparar un pedido.

## Usuario

**Picker / Operario** (`operario`)

## Descripción

Como operario, quiero ver una lista de picking con productos, cantidades y ubicaciones ordenadas por pasillo/estante/posición, para preparar el pedido de salida recorriendo el almacén de forma lógica.

## Criterios de aceptación

- [ ] Al iniciar picking, el sistema genera `picking_lines` asignando ubicaciones con stock disponible
- [ ] Lista ordenada por `aisle`, `rack`, `position`
- [ ] Cada línea muestra: producto (SKU, nombre), ubicación, cantidad a pickear, unidad
- [ ] Si un producto requiere varias ubicaciones, se dividen en múltiples picking lines
- [ ] Muestra progreso: líneas pendientes vs confirmadas
- [ ] Búsqueda de producto en vista por SKU o nombre

## Reglas de negocio

1. Estrategia de asignación MVP: **FIFO por ubicación** (orden alfabética de ubicación, consumir primera con stock).
2. Solo ubicaciones del almacén de la orden.
3. Cantidad a pickear ≤ stock disponible en ubicación (`on_hand - reserved` propio + reserva de esta orden).
4. No se genera picking para orden en `draft`.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Stock en una sola ubicación insuficiente | Dividir entre ubicaciones o marcar línea pendiente |
| Ubicación desactivada con stock legacy | Excluir de nuevas asignaciones |
| Regenerar picking tras cambio stock | Solo si ninguna línea confirmada; sino manual |
| Producto en múltiples ubicaciones mismo pasillo | Ordenar por position |

## Notas técnicas preliminares

- Modelo `PickingLine`: `outbound_line_id`, `location_id`, `quantity_to_pick`, `quantity_picked`, `sequence`, `status`
- Servicio `Picking::GenerateLines.call(outbound_order)`
- Algoritmo greedy por ubicación ordenada
- ADR recomendado: estrategia de asignación y reservas
- Rama: `feature/warehouse-picking`
