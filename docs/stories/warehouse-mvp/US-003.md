# US-003: Gestión de almacenes

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** ninguna

---

## Objetivo

Registrar los centros de distribución/almacenes de la comercializadora para segmentar inventario y operaciones por sede.

## Usuario

**Administrador** (`admin`)

## Descripción

Como administrador, quiero crear y gestionar almacenes con código y nombre, para operar stock y movimientos de forma independiente en cada centro de distribución.

## Criterios de aceptación

- [ ] Puedo crear un almacén con código único y nombre
- [ ] Puedo registrar dirección opcional
- [ ] Puedo desactivar un almacén
- [ ] Almacén inactivo no permite nuevas recepciones, salidas ni transferencias
- [ ] Almacén inactivo sí permite consulta de stock e historial
- [ ] El listado muestra código, nombre, estado y stock total de productos (resumen)

## Reglas de negocio

1. Código de almacén único (ej. `CD-MAD`, `CD-BCN`).
2. Mínimo 1 almacén activo en el sistema en todo momento.
3. Desactivar no elimina datos históricos ni stock registrado.
4. Operarios pueden estar asignados a un almacén por defecto (configuración P1; en MVP filtran manualmente).

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Desactivar único almacén activo | Rechazado |
| Desactivar almacén con transferencias en tránsito | Rechazado hasta completar o cancelar |
| Código con minúsculas | Normalizar a mayúsculas al guardar |
| Reactivar almacén | Permitido; operaciones nuevas habilitadas |

## Notas técnicas preliminares

- Modelo `Warehouse`: `code`, `name`, `address`, `active`
- Índice único en `code`
- `ExternalReference` preparado para mapeo Odoo `stock.warehouse`
- API: `GET/POST/PATCH /api/warehouses`
- Política: solo `admin` puede crear/editar almacenes
