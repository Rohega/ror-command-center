# US-041: Historial de movimientos de inventario

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-005

---

## Objetivo

Investigar discrepancias y auditar operaciones consultando el registro inmutable de cambios de stock por producto.

## Usuario

**Supervisor** (`supervisor`), **Administrador** (`admin`), **Consulta** (`consulta` — solo lectura)

## Descripción

Como supervisor, quiero consultar el historial de movimientos de un producto filtrando por almacén, tipo y rango de fechas, para investigar diferencias y cumplir trazabilidad operativa.

## Criterios de aceptación

- [ ] Listado paginado de `stock_movements` filtrable por: producto, almacén, tipo, usuario, rango fechas
- [ ] Cada registro muestra: fecha, tipo, cantidad (+/-), antes/después, ubicación, usuario, referencia (orden origen), notas
- [ ] Movimientos cancelados muestran enlace a reversión y badge "cancelado"
- [ ] Export CSV del filtro actual
- [ ] Registros son de solo lectura en UI (sin edición)
- [ ] Ordenación por `occurred_at` descendente

## Reglas de negocio

1. `stock_movements` append-only; sin update/delete desde aplicación.
2. Toda mutación vía `StockUpdater` genera exactamente un movimiento (salvo operaciones multi-ubicación: un movimiento por línea).
3. Consulta puede ver todo el historial; no export restringido en MVP.
4. Retención: ilimitada en MVP.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| Producto sin movimientos | Lista vacía con mensaje |
| Movimiento de transferencia | Mostrar par out/in vinculado visualmente (P1); MVP: tipo y referencia |
| Alto volumen (>100k movimientos) | Paginación + índices; sin carga total |
| Filtro fecha inválida | Validación |

## Notas técnicas preliminares

- Tabla `stock_movements` con índices `(product_id, occurred_at)`, `(warehouse_id, occurred_at)`
- `reference_type`/`reference_id` polimórfico
- API: `GET /api/stock_movements?product_id=&warehouse_id=&...`
- UI en `/products/:id/movements` y vista global `/audit/movements`
- Rama: `feature/warehouse-stock-core` (API) + `feature/warehouse-audit-ui` (UI completa)
