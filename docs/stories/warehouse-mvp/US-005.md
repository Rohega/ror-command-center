# US-005: Carga de stock inicial

**Spec:** [warehouse-mvp](../../specs/warehouse-mvp.md)  
**Prioridad:** P0  
**Dependencias:** US-002, US-004

---

## Objetivo

Establecer el inventario inicial en cada almacén al arrancar el sistema en blanco, sin integración ERP.

## Usuario

**Administrador** (`admin`)

## Descripción

Como administrador, quiero cargar cantidades iniciales de productos por ubicación y almacén, mediante formulario individual o importación CSV, para comenzar a operar con stock real desde el día uno.

## Criterios de aceptación

- [ ] Puedo asignar cantidad inicial a combinación producto + ubicación
- [ ] La carga crea o actualiza `stock_level` y registra `stock_movement` tipo `adjustment` con nota "stock inicial"
- [ ] Puedo importar CSV con columnas: `warehouse_code`, `location_code`, `sku`, `quantity`
- [ ] El import reporta filas válidas e inválidas sin aplicar parcialmente un lote inválido (todo-o-nada por archivo)
- [ ] Cantidad debe ser ≥ 0
- [ ] Producto y ubicación deben existir y estar activos
- [ ] Ubicación debe pertenecer al almacén indicado

## Reglas de negocio

1. Carga inicial solo permitida si `quantity_on_hand` actual es 0 para esa combinación, salvo re-import explícito con flag `overwrite` (solo admin).
2. Toda carga pasa por `StockUpdater`.
3. Unidades según `unit_type` del producto; sin conversión.
4. Movimiento de auditoría con `user_id` del admin que carga.

## Casos borde

| Caso | Comportamiento esperado |
|------|-------------------------|
| SKU inexistente en CSV | Error en fila; rollback del import completo |
| Cantidad decimal en producto `unidad` | Permitir decimales solo para `kg` y `litro`; enteros recomendados para resto |
| Misma combinación duplicada en CSV | Rechazar archivo con error de duplicado |
| Ubicación de otro almacén | Rechazar fila |
| Stock ya existente sin overwrite | Rechazar fila con mensaje |

## Notas técnicas preliminares

- Servicio `StockUpdater#apply_initial_stock(product, location, quantity, user)`
- Job opcional para imports grandes (>1000 filas)
- Template CSV descargable desde UI
- Transacción DB por archivo de import
- Rama: `feature/warehouse-stock-core`
